# 电商智能体演示系统 - 需求文档

## 1. 项目概述

本项目旨在构建一个智能体演示系统，该智能体可以与电商业务系统进行交互，实现用户管理、商品管理、订单管理等核心功能。系统支持商家和买家两种角色，提供完整的电商业务流程演示。

### 1.1 核心功能
- **用户管理**：支持用户切换、用户详情展示
- **商品管理**：商家可上架、新增、修改、删除商品
- **订单管理**：买家可使用余额购买商品，余额不足时提示
- **智能体交互**：通过大模型实现自然语言交互
- **语音交互**：支持语音输入（语音转文字）

## 2. 功能需求

### 2.1 用户管理功能

#### 2.1.1 用户切换
- **功能描述**：Demo演示界面支持点击选择切换用户
- **交互方式**：前端提供用户列表，点击切换当前登录用户
- **显示内容**：用户详情界面展示用户信息

#### 2.1.2 用户信息管理
- **用户字段**：
  - 用户名（唯一标识）
  - 角色（商家/买家）
  - 账户余额（仅买家需要）
- **操作**：
  - 查看用户详情
  - 切换当前用户身份

### 2.2 商品管理功能（商家）

#### 2.2.1 商品上架
- **功能描述**：商家可以选择上架商品信息
- **操作流程**：商家登录后，可添加新商品到系统

#### 2.2.2 商品信息管理
- **商品字段**：
  - 商品名称
  - 商品描述
  - 单价
  - 库存数量
- **操作**：
  - 新增商品
  - 修改商品信息
  - 删除商品
  - 查看商品列表

### 2.3 订单管理功能（买家）

#### 2.3.1 购买商品
- **功能描述**：买家可以使用账户余额购买商品
- **业务流程**：
  1. 买家浏览商品列表
  2. 选择商品并指定购买数量
  3. 系统计算总金额
  4. 检查账户余额是否充足
  5. 余额充足：创建订单，扣除余额，减少商品库存
  6. 余额不足：提示用户余额不足，无法完成购买

#### 2.3.2 订单信息
- **订单字段**：
  - 买家的 User ID
  - 商品 ID
  - 购买数量
  - 交易总金额
  - 下单时间

### 2.4 智能体交互功能

#### 2.4.1 自然语言交互
- **功能描述**：通过大模型实现智能体与系统的自然语言交互
- **交互场景**：
  - 查询商品信息
  - 查询订单信息
  - 查询用户信息
  - 执行购买操作
  - 执行商品管理操作

#### 2.4.2 语音输入（语音转文字）
- **功能描述**：用户可以通过语音输入与智能体交互
- **实现方式**：
  - 使用浏览器 Web Speech API（SpeechRecognition）
  - 支持实时语音识别，将语音转换为文字
  - 识别结果自动填入对话输入框
- **交互流程**：
  1. 用户点击麦克风按钮开始录音
  2. 系统实时识别语音并显示识别结果
  3. 用户确认或修改识别结果
  4. 提交文字内容到智能体
- **功能特性**：
  - 支持开始/停止录音控制
  - 实时显示识别状态（正在识别/识别完成）
  - 支持识别结果编辑
  - 错误提示（如麦克风权限被拒绝）

#### 2.4.3 语音输出（文字转语音）
- **功能描述**：智能体的回复可以通过语音播放
- **实现方式**：
  - 使用浏览器 Web Speech API（SpeechSynthesis）
  - 将智能体返回的文字内容转换为语音播放
- **交互流程**：
  1. 智能体返回文字回复
  2. 用户可选择是否启用语音播放
  3. 系统自动将文字转换为语音并播放
- **功能特性**：
  - 支持语音播放开关（可开启/关闭）
  - 支持暂停/继续播放
  - 支持语速、音调、音量调节
  - 支持选择不同的语音引擎和语言
  - 自动跳过标点符号和特殊字符

#### 2.4.4 常用提问示例

智能体支持自然语言交互，以下是一些常用的提问示例：

**商品查询类**：
- "帮我查询一下现在有哪些商品"
- "有哪些商品？"
- "显示所有商品"
- "商品ID:1 的信息是什么？"
- "查询iPhone手机的信息"

**购买商品类**：
- "我需要购买智能机械键盘"
- "帮我购买一件耳机"
- "我要买iPhone手机"
- "购买商品ID:3，数量2件"
- "买一个索尼耳机"

**订单查询类**：
- "查询一下我购买了哪些商品"
- "我的订单有哪些？"
- "显示我的订单列表"
- "查看订单ID:1的详情"

**用户信息类**：
- "我的余额是多少？"
- "查询我的账户信息"
- "我的用户信息"

**组合查询类**：
- "帮我查询一下现在有哪些商品" → "我需要购买智能机械键盘" → "查询一下我购买了哪些商品"
  - 智能体会记住上下文，第二句中的"智能机械键盘"会匹配到第一句查询结果中的商品
  - 第三句查询订单时，会显示刚才购买的订单信息

**注意事项**：
- 智能体支持对话记忆，能够理解上下文
- 购买商品时，如果只提到商品名称，系统会自动匹配商品
- 如果余额不足或库存不足，系统会明确提示失败原因
- 切换用户时，聊天记录会自动清空

## 4. API 接口设计

### 4.1 用户相关接口

#### 4.1.1 获取用户列表
- **接口**：`GET /api/users`
- **说明**：获取所有用户列表，用于用户切换
- **响应**：用户列表（包含id、username、role、balance）

#### 4.1.2 获取用户详情
- **接口**：`GET /api/users/{user_id}`
- **说明**：获取指定用户的详细信息
- **响应**：用户详细信息

#### 4.1.3 切换当前用户
- **接口**：`POST /api/users/switch`
- **参数**：`{"user_id": 1}`
- **说明**：切换当前登录用户（前端状态管理）

### 4.2 商品相关接口

#### 4.2.1 获取商品列表
- **接口**：`GET /api/products`
- **说明**：获取所有商品列表
- **响应**：商品列表

#### 4.2.2 获取商品详情
- **接口**：`GET /api/products/{product_id}`
- **说明**：获取指定商品的详细信息
- **响应**：商品详细信息

#### 4.2.3 新增商品（商家）
- **接口**：`POST /api/products`
- **参数**：`{"name": "商品名", "description": "描述", "price": 99.9, "stock": 100, "seller_id": 1}`
- **说明**：商家新增商品

#### 4.2.4 修改商品（商家）
- **接口**：`PUT /api/products/{product_id}`
- **参数**：`{"name": "新名称", "description": "新描述", "price": 89.9, "stock": 90}`
- **说明**：商家修改商品信息

#### 4.2.5 删除商品（商家）
- **接口**：`DELETE /api/products/{product_id}`
- **说明**：商家删除商品

### 4.3 订单相关接口

#### 4.3.1 创建订单（买家）
- **接口**：`POST /api/orders`
- **参数**：`{"buyer_id": 2, "product_id": 1, "quantity": 2}`
- **说明**：买家创建订单，系统自动计算总金额、检查余额和库存
- **响应**：成功返回订单信息，失败返回错误信息（余额不足/库存不足）

#### 4.3.2 获取订单列表
- **接口**：`GET /api/orders`
- **参数**：`?buyer_id=2`（可选，筛选买家订单）
- **说明**：获取订单列表

#### 4.3.3 获取订单详情
- **接口**：`GET /api/orders/{order_id}`
- **说明**：获取指定订单的详细信息

### 4.4 智能体交互接口

#### 4.4.1 智能体对话
- **接口**：`POST /api/agent/chat`
- **参数**：`{"message": "用户输入的自然语言", "user_id": 1}`
- **说明**：通过大模型处理用户自然语言请求，执行相应操作
- **响应**：智能体的回复和操作结果

### 5.2 交互设计

#### 5.2.1 用户切换
- **实现方式**：点击用户列表项，更新当前用户状态
- **状态管理**：前端使用 JavaScript 变量存储当前用户ID
- **界面更新**：切换用户后，自动刷新相关数据展示

#### 5.2.2 余额不足提示
- **触发条件**：买家下单时，账户余额 < 订单总金额
- **提示方式**：弹窗或页面提示信息
- **提示内容**：余额不足，当前余额：XX，订单金额：XX

#### 5.2.3 语音输入交互
- **实现方式**：
  - 点击麦克风按钮开始录音
  - 显示录音状态指示器（动画效果）
  - 实时显示识别结果（可选）
  - 再次点击停止录音并确认识别结果
- **状态管理**：
  - 录音中：显示红色录音指示器
  - 识别中：显示识别动画
  - 识别完成：显示识别结果，可编辑
- **错误处理**：
  - 麦克风权限被拒绝：提示用户授权
  - 识别失败：提示重试
  - 网络错误：提示检查网络连接

#### 5.2.4 语音输出交互
- **实现方式**：
  - 语音输出开关：全局控制是否启用语音播放
  - 每条消息可单独控制播放/暂停
  - 播放时显示播放进度指示器
- **控制功能**：
  - 播放：点击播放按钮开始播放
  - 暂停：点击暂停按钮暂停播放
  - 停止：点击停止按钮停止播放
  - 语速调节：滑块控制播放速度（0.5x - 2.0x）
  - 音量调节：滑块控制播放音量（0 - 100%）
- **用户体验**：
  - 自动播放：智能体回复后自动播放（如果启用）
  - 播放队列：多条消息按顺序播放
  - 播放状态：清晰显示当前播放状态

### 6.1 后端技术栈

## 9. 注意事项

1. **安全性**：
   - 环境变量中的密钥不能提交到代码仓库
   - 前端操作需要验证用户权限（商家只能管理自己的商品）

2. **数据一致性**：
   - 订单创建时，需要事务处理（扣除余额、减少库存）
   - 余额和库存检查需要在数据库层面保证原子性

3. **用户体验**：
   - 操作反馈要及时（成功/失败提示）
   - 余额不足时给出明确的提示信息
   - 界面要简洁清晰，易于操作
   - 语音交互要流畅自然，提供清晰的状态反馈

4. **语音功能注意事项**：
   - **浏览器兼容性**：
     - SpeechRecognition API 主要在 Chrome/Edge 浏览器支持
     - SpeechSynthesis API 主流浏览器均支持
     - 需要提供降级方案（不支持时隐藏语音功能）
   - **权限管理**：
     - 首次使用需要请求麦克风权限
     - 权限被拒绝时提供友好的提示和引导
   - **性能优化**：
     - 语音识别结果实时显示，避免阻塞主线程
     - 语音播放支持中断和队列管理
     - 长时间录音时注意内存管理
   - **用户体验**：
     - 提供清晰的录音状态指示（动画、颜色变化）
     - 识别结果可编辑，允许用户修正错误
     - 语音播放支持暂停、继续、停止操作
     - 支持语速和音量调节

## 6. 技术栈与规范

### 6.1 后端技术栈

| 项目     | 选型                     | 说明                                     |
| -------- | ------------------------ | ---------------------------------------- |
| Web 框架 | FastAPI                  | 异步、自动 OpenAPI 文档                  |
| 数据库   | SQLite3                  | 单文件、无需独立服务                     |
| 环境配置 | `.env` + `python-dotenv` | 密钥与配置不入库                         |
| 大模型   | 火山引擎 Ark             | `volcenginesdkarkruntime`，配置见 `.env` |
| MCP 框架 | FastMCP                  | 可选，用于工具调用，参考 `test/mcp-demo` |
| 提示词管理 | PromptManager            | 统一管理所有提示词模板，位于 `backend/prompts.py` |

### 6.2 前端技术栈

| 项目 | 选型                  | 说明                    |
| ---- | --------------------- | ----------------------- |
| 页面 | 原生 HTML             | 不引入 Vue/React 等框架 |
| 样式 | 原生 CSS 或单文件内联 | 保持简单                |
| 交互 | 原生 JavaScript       | `fetch` 调用后端 API    |
| 语音识别 | Web Speech API (SpeechRecognition) | 浏览器原生语音转文字 API |
| 语音合成 | Web Speech API (SpeechSynthesis) | 浏览器原生文字转语音 API |

### 6.3 核心特性

- **对话记忆**：智能体支持保留最近10轮对话历史，能够理解上下文
- **实际执行**：智能体可以真正执行购买等操作，不仅仅是回复文本
- **提示词统一管理**：所有提示词集中管理在 `backend/prompts.py`，便于维护和修改
- **MCP 工具支持**：可选使用 MCP 框架进行工具调用，提供更灵活的功能扩展
